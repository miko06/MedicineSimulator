# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## Медицинский тренажёр «MedSim» — платформа клинической симуляции

**Версия:** 0.1.0  
**Дата:** 10 июня 2026  
**Тип:** Веб-приложение (SPA + REST API)

## 1. ОБЩЕЕ ОПИСАНИЕ

### 1.1 Назначение

MedSim — интерактивная платформа для студентов медицинских вузов, позволяющая практиковать диагностические навыки на виртуальном роботе-пациенте. Платформа моделирует клинический приём: студент изучает симптомы, осматривает 3D-модель тела с подсвеченными зонами поражения, слушает голосовые жалобы пациента через синтез речи и ставит диагноз. Система оценивает ответ и показывает правильный диагноз с планом лечения.

Платформа также включает админ-панель для преподавателей, позволяющую создавать и редактировать упражнения, управлять пользователями и отслеживать статистику успеваемости.

### 1.2 Целевая аудитория

- **Студенты медицинских вузов** — проходят клинические симуляции по разделам (хирургия, кардиология, неврология и др.)
- **Преподаватели/администраторы** — создают учебные материалы, отслеживают прогресс студентов

### 1.3 Языки

Платформа трёхъязычная:

- 🇬🇧 English (EN)
- 🇷🇺 Русский (RU)
- 🇰🇿 Қазақша (KZ)

Переключение — кнопка в шапке. Контент (названия, описания, диагнозы) хранится во всех трёх языках. Голосовые жалобы пациента озвучиваются на русском языке с возможностью расширения.

## 5. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 5.1 Аутентификация и авторизация

**Регистрация**

- Студент регистрируется по email + пароль (мин. 6 символов)
- Пароль хешируется bcrypt (12 раундов)
- После регистрации автоматически выполняется вход

**Вход**

- Проверка email + пароль
- Выдаются два JWT-токена в httpOnly cookies:
  - `access_token` — 15 минут, путь `/`
  - `refresh_token` — 30 дней, путь `/api/auth`

**Обновление токена**

- `POST /api/auth/refresh` — проверяет `refresh_token`, выдаёт новый `access_token`

**Роли**

- `STUDENT` — доступ к упражнениям, прогрессу, профилю
- `ADMIN` — дополнительный доступ к панели управления

**Защита**

- Rate-limit: 10 запросов в минуту на IP для `/register` и `/login`
- Нельзя изменить свою роль, деактивировать или удалить свой аккаунт

### 5.2 Упражнения и клиническая симуляция

**Структура упражнения**

Каждое упражнение содержит:

1. **Сценарий пациента** — текстовое описание жалоб и анамнеза (на 3 языках)
2. **Симптомы** (2-4 шт.) — название, зона тела, степень тяжести (1-10), цвет подсветки, описание
3. **3D-пресет робота** — массив `zoneOverrides` с зонами, цветом и интенсивностью подсветки
4. **Диагнозы** — правильный диагноз (с планом лечения) + 1-2 неправильных
5. **Снимки пациента** — массив путей к изображениям (рентген, МРТ, фото)
6. **Голос пациента** — JSON `{text, voiceId}` для синтеза речи через ElevenLabs

**Процесс прохождения**

1. Студент открывает упражнение
2. Видит 3D-робота с подсвеченными красным зонами поражения
3. Кликает по зонам тела — фильтрует список симптомов
4. Просматривает снимки пациента
5. Слушает голосовые жалобы (кнопка «Listen to patient»)
6. Нажимает «Start Diagnosis»
7. Выбирает диагноз из предложенных вариантов
8. Получает обратную связь:
   - ✅ Правильно — поздравление, детали диагноза
   - ❌ Неправильно — показывается правильный диагноз с планом лечения
9. Результат сохраняется в историю попыток

**Список упражнений**

- Фильтрация по разделу (специальности), сложности, поиск по названию
- Пагинация (12 упражнений на страницу)
- Отображение: название, описание, сложность, раздел, количество симптомов/диагнозов/попыток

### 5.3 3D-робот

**Два режима:**

1. **Spline-сцена** — загружается внешняя 3D-модель (`@splinetool/react-spline`). Объекты тела подсвечиваются программно через `spline.findObjectByName()`. Поддерживает клики и наведение.

2. **Процедурная модель** (Three.js/R3F) — полностью сгенерированный человекоподобный робот из геометрических примитивов. Анимация дыхания, парящие частицы, тени.

**Взаимодействие:**

- Клик по зоне тела → фильтрация симптомов
- Наведение → wireframe-обводка зоны
- Подсветка симптомов: цвет + пульсирующее свечение (интенсивность зависит от severity)
- OrbitControls: вращение, зум (без панорамирования)

### 5.4 Голосовые жалобы (ElevenLabs TTS)

- Бэкенд-эндпоинт: `POST /api/exercises/:id/voice`
- API-ключ ElevenLabs в `.env`
- Параметры: `voiceId`, `text`, `modelId: eleven_multilingual_v2`
- Возвращает аудио в формате `audio/mpeg`
- Фронтенд: кнопка «▶ Listen to patient», загрузка с индикатором, стоп

### 5.5 Снимки пациента

- Хранятся в `webapp/public/images/`
- Загрузка через админ-панель (drag-and-drop, множественный выбор файлов)
- Просмотр: галерея с навигацией Prev/Next, фуллскрин по клику
- Удаление: крестик при наведении в админке

### 5.6 Статистика и прогресс

**Студент:**

- Сводка: всего попыток, завершено, средний балл, лучший балл, общее время
- По разделам: количество, завершено, средний балл
- Графики (Recharts):
  - Круговая диаграмма — распределение упражнений по разделам
  - Столбчатая диаграмма — успеваемость по разделам
  - Линейный график — тренд последних оценок
- История попыток с фильтрами (статус, раздел) и пагинацией
- Таблица лидеров (средний балл, количество завершённых)
- Экспорт в CSV

**Администратор:**

- Общая статистика платформы: студенты, попытки, % завершения, средний балл
- Активность по дням (30 дней)
- Просмотр прогресса любого студента

### 5.7 Админ-панель

**Управление контентом:**

| Сущность | Создание | Чтение | Редактирование | Удаление |
|----------|----------|--------|----------------|----------|
| Разделы (Specialties) | ✅ | ✅ | ✅ | ✅ |
| Симптомы (Symptoms) | ✅ | ✅ | ✅ | ✅ |
| Диагнозы (Diagnoses) | ✅ | ✅ | ✅ | ✅ |
| Упражнения (Exercises) | ✅ | ✅ | ✅ | ✅ |
| Пресеты робота | ✅ | ✅ | ✅ | ✅ |

**Управление пользователями:**

- Список с пагинацией (20 на страницу)
- Смена роли (STUDENT ↔ ADMIN)
- Блокировка/разблокировка (isActive)
- Удаление
- Просмотр прогресса студента

**Дашборд:**

- Карточки: студенты, админы, упражнения, средний балл
- Таблица упражнений по разделам
- Последняя активность (5 записей)

### 5.8 Профиль пользователя

- Просмотр: email, роль
- Редактирование: отображаемое имя (displayName), учреждение (institution)

### 5.9 Интернационализация (i18n)

- Три языка: EN, RU, KZ
- Переключатель в шапке (EN → RU → KZ → EN)
- Все сущности имеют поля для трёх языков (nameEn/Ru/Kz, descriptionEn/Ru/Kz, titleEn/Ru/Kz)
- Фоллбэк: KZ → RU → EN
- Локаль сохраняется в localStorage
- API-клиент отправляет `Accept-Language` во всех запросах

### 5.10 Тема оформления

- Две темы: тёмная (по умолчанию) и светлая
- Переключатель в шапке (иконка солнце/луна)
- CSS-переменные для цветов (bg, surface, border, muted, text, accent)
- Tailwind с поддержкой opacity-модификаторов через RGB-переменные
- Glass-эффект (backdrop-filter)

## 7. БЕЗОПАСНОСТЬ

1. **JWT в httpOnly cookies** — токены недоступны из JavaScript
2. **bcrypt** — 12 раундов хеширования паролей
3. **Rate-limiting** — 10 запросов/минуту на auth-эндпоинты
4. **Защита от self-action** — админ не может изменить/удалить/деактивировать себя
5. **CORS** — белый список разрешённых origin
6. **API-ключ ElevenLabs** — хранится только в `.env` бэкенда, не передаётся на фронтенд
7. **Проверка ролей** — middleware `requireAuth` и `adminGuard` на каждом защищённом эндпоинте
